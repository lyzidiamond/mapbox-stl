Part 1: Sources and styles
All Mapbox maps are made from a combination of styles and sources.
The source is the underlying vector data that makes up the features on the map.
(Sources can also be raster images, but we will come back to that.)
The style is a file with a list of rules for how the source should be rendered.
Mapbox Studio has a source mode and a style mode, and we're going to go over both...
... but to understand and appreciate why the style-source paradigm is effective, we need to learn about map tiles and how they have changed over time.
The biggest challenge in web mapping from the very beginning has been speed.
The world is a big place, and loading detailed images of the whole thing takes time.
Thus came the invention of the map tile from Google in 2005.
Map tiles were 256px x 256px PNG files that were stored on the server and served to the browser on request.
All of these small images load way faster than one giant image...
... and tiles are only loaded where you're looking in the world and at that one zoom level.
This was way faster than anything else.
How were these tiles made?
At Mapbox, we used a tool called TileMill.
- Start with vector data (shapefile, GeoJSON, etc) and load into TileMill
- Create a stylesheet using CartoCSS, a styling language, in TileMill
- Export the data and stylesheet from TileMill to Mapbox.com, where a renderer (Mapnik) creates map tiles
The images were created at each zoom level and stored in a specific file structure as PNGs.
When your map needs the image tile for a particular area at a particular zoom, your browser requests it from the server.
This worked super well for a while, but had some drawbacks, particularly with retina displays.
Storing retina tiles AND non-retina tiles would take up lots of space!
And storing images was taking up a lot of space in the first place.
So instead of storing fully-rendered images...
... we started storing the source data separately from the style rules and then generating images when requested.
It's a little confusing, but fortunately Mapbox Studio makes it fairly straightforward!
- Start with vector data (shapefile, GeoJSON, etc) and load into Mapbox Studio source mode
- Upload the vector data to Mapbox.com where it is converted to a vector tile dataset (PBF)
- Load that vector tile dataset into Mapbox Studio style mode and create a stylesheet using CartoCSS to style the data
- Upload the style to Mapbox.com.
The style now has a reference to the source.
Now, when your map needs the image tile for a particular area at a particular zoom, your browser requests it from the server...
... and the server applies the style to the source, generates an image (with Mapnik), and serves that image to the client.
We call these vector tiles.
Vector tiles are awesome because:
- All of the properties and geometries of the vector data are stored.
- Multiple styles can be applied to the same source data
- The info takes up way less space, which means it's way less expensive
- It opens up the door for different rendering options like GL
(More about that in a bit.)
So there you have it: sources and styles.
One of the awesome things about Mapbox Studio is that it comes with a couple sources: Mapbox Streets and Mapbox Terrain.
Today we will use these sources and focus on styling with CartoCSS.
CartoCSS is the language used in Mapbox Studio to style vector tile data.
At its core it is a pre-processing language for Mapnik, the tool that renders the tiles, that is easy enough for us to use and machine-like enough for Mapnik to understand.

Part 2: Quick notes on the new Mapbox Studio
As some of you may have seen, we are getting ready to release a new version of Mapbox Studio:
In the new Mapbox Studio, there will still be sources and styles.
Sources will stay the same, but it's the styling that will be different.
The new Mapbox Studio will rely on client-side GL styling instead of server-side Mapnik styling.
This will make the maps even faster and allow things like fractional zoom styling, map rotation, and more freedom outside the overlay/basemap way of thinking (all vectors!)
Just like CartoCSS is a way for us to create Mapnik styles, the new Mapbox Studio will be a way for us to create GL styles.
This will be a bit different, but by understanding how sources work, you will already have a leg up!
